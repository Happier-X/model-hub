# v0.0.1 (2026-07-23)

Model Hub 首发：本机 Vue 3 管理台 + Tauri 进程内 Rust 代理，统一 Base URL，按分组故障转移转发 OpenAI 兼容 Chat。

## 核心能力

- 多供应商配置（上游 API Key 仅存于供应商配置）
- 分组名 = 客户端 `model`，组内有序故障转移队列
- 默认熔断（连续失败阈值 / 恢复等待 / 半开）
- 本机 `/v1/*` 无客户端 API Key 鉴权（有无 `Authorization` 均放行）
- `POST /v1/chat/completions`（非流式 + SSE）、`GET /v1/models`
- Windows NSIS 安装包与 Tauri Updater 签名更新（`latest.json`）

## 管理台

- 供应商、分组使用对话框新建 / 编辑；编辑使用稳定实体 id，避免误创建重复项
- 供应商支持粘贴快速识别 Base URL 与 API Key
- 分组队列支持拖拽排序、本地启发式与 OpenRouter 公共榜单混合排序
- 分组页「拉取模型」仅用户点击触发，不自动测活上游
- 分组页一键「配置到 Pi」：按分组名 upsert 写入本机 `~/.pi/agent/models.json` 的 `model-hub`（固定占位 Key）
- 无客户端 API Key 管理页面；健康状态只读展示，无手动刷新入口

## 运行与运维

- 代理首选端口被占用时自动选择可用端口并持久化
- 关闭窗口隐藏到托盘，代理继续运行；仅托盘「退出」停止代理并释放端口
- 单实例插件，避免多开抢占端口
- 请求日志默认保留 30 天，日志页可定时刷新
- 概览页支持手动检查更新与可选启动检查

## 安装与更新

- 安装包：GitHub Release 中的 NSIS `.exe`
- 校验：同 Release 的 `SHA256SUMS.txt`
- 应用内更新清单：`https://github.com/Happier-X/model-hub/releases/latest/download/latest.json`
- 客户端 Base URL 使用概览页本机地址；`model` 填分组名
