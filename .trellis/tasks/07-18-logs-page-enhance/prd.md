# 日志页补强

## Goal

提升日志页排障体验：支持**分页**、**本页过滤**（模型/渠道/仅错误）、**行详情展开**，以及清空确认；在侧车仅提供 list/clear 能力的前提下最大化可用性。

## Background

- 当前：固定 `listLogs(1, 40)`、5s 轮询、表格字段有限、清空无确认、错误列截断。
- 侧车 v0.9.28：`GET /api/v1/log/list?page&page_size&start_time&end_time`（page_size 上限 100）；`DELETE /api/v1/log/clear`。
- **无**服务端按 model/error 过滤参数 → 过滤在**当前已加载页**客户端完成，并在 UI 标明。

## Requirements

### R1. 分页

- 页码 + 每页条数（默认 20 或 30，可选 20/50/100，不超过侧车上限）。
- 上一页/下一页；当本页条数 `< pageSize` 时禁用「下一页」（无 total 时的近似）。
- 切换分页时重新请求；可选保留自动刷新（仅刷新当前页）。

### R2. 过滤（客户端，当前页）

- 关键词：匹配 `request_model_name` / `actual_model_name` / `channel_name` / `error`（不区分大小写包含）。
- 开关：「仅显示有错误」。
- UI 提示：过滤作用于**当前页数据**，非全库检索。

### R3. 详情展开

- 行点击或「详情」展开：展示完整字段（时间、请求模型、实际模型、渠道、token、耗时、费用、完整 error 等已有字段）。
- 错误长文本可换行/复制。

### R4. 清空与刷新

- 清空前确认。
- 手动刷新；自动轮询可开关（默认开，间隔约 5s），避免排障时列表跳动可选关。

### R5. 约束

- 不接 SSE stream（本任务不做实时推送）。
- 不改侧车二进制。
- 不引入后端代理日志存储。

## Out of Scope

- 服务端全文搜索 / 导出 CSV。
- 日志图表统计。
- SSE `/api/v1/log/stream`。

## Acceptance Criteria

- [x] AC1：可切换页码/每页条数并加载对应 list。
- [x] AC2：关键词与「仅错误」过滤作用于当前页，并有说明文案。
- [x] AC3：可展开查看单条完整字段与完整 error。
- [x] AC4：清空有确认；自动刷新可关。
- [x] AC5：`pnpm lint` / `pnpm build` 通过；既有 smoke 不回归。
