# Windows 托盘与关闭行为

## Goal

把 Model Hub 从「关闭窗口即退出」升级为常见 Windows 桌面行为：支持系统托盘菜单；点击窗口关闭按钮默认**隐藏到托盘**而不是立刻停止网关；只有托盘「退出」或应用真正退出时才停止托管侧车。

## Background

- 当前 Tauri `RunEvent::Exit` 会停止网关，但没有托盘入口。
- `src-tauri/Cargo.toml` 已有 Tauri 2；权限 schema 包含 core tray。
- 当前壳已有自动启动侧车（`try_autostart`）。

## Requirements

### R1. 系统托盘

- 创建托盘图标（复用 `src-tauri/icons/icon.ico` 或 Tauri 可接受格式）。
- 菜单至少包含：
  - 「显示 Model Hub」
  - 「启动/停止网关」（至少一个状态相关动作，或先提供显示/退出）
  - 「退出」
- 左键/双击托盘图标显示并聚焦主窗口。
- 菜单「显示」显示并聚焦；「退出」真正退出应用。

### R2. 关闭行为

- 主窗口 `CloseRequested` 默认 `prevent_close` + `hide`，不触发 `RunEvent::Exit`，不停止网关。
- 用户从托盘选择「退出」时允许退出，最终触发已有 `stop_managed`。
- 隐藏后可从托盘恢复；不产生重复窗口/托盘实例。

### R3. 用户提示

- 首次或每次关闭时可通过托盘 tooltip/菜单文案说明：关闭窗口会隐藏，托盘退出才停止网关。
- 设置或状态提示补充该行为（中文）。

### R4. 生命周期与失败处理

- 托盘创建失败不得静默导致编译/启动不可用；给出可诊断错误。
- 退出路径仍只停止**本应用托管**的 GatewayHandle，不按进程名杀 octopus。
- 不新增后台全局进程。

### R5. 可选开机自启

- 本任务只做规划，不强制实现开机自启；若 Tauri 插件未引入则不添加依赖。

## Out of Scope

- 开机自启设置。
- 最小化到托盘以外的窗口布局改造。
- 托盘实时菜单状态动态化（可后续增强）。
- macOS/Linux 正式验收。

## Acceptance Criteria

- [x] AC1：Windows 构建/运行后存在 Model Hub 托盘图标与显示/退出菜单。
- [x] AC2：点击关闭窗口后窗口隐藏，托管网关仍运行；托盘显示可恢复。
- [x] AC3：托盘退出后应用退出且托管网关被停止。
- [x] AC4：`cargo test` / `cargo check` 通过；前端 lint/build 通过。
- [x] AC5：文案明确「关窗隐藏、托盘退出才停网关」，不影响已有网关测试策略。
