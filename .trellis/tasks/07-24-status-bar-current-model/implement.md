# 执行计划：桌面悬浮状态条显示当前模型

## 前置门禁

- [ ] 加载 backend/frontend Trellis 规范并确认本任务涉及的目录约定。
- [ ] 确认与当前 Tauri 2.11.5 兼容的 `tauri-plugin-window-state` 固定版本。
- [ ] 验证插件是否可按窗口 label 排除 `main`；若不可，采用仅对 `overlay` 手动调用位置保存/恢复，禁止改变主窗口现有状态。
- [ ] 确认 overlay 窗口所需 capability：至少允许 invoke、窗口拖动、位置读取及手动 window-state 操作；权限只授予 `overlay`。

## 实现步骤

1. **扩展壳配置**
   - 在 `ShellConfig` 增加 `overlay_enabled: bool`，serde 默认 false。
   - 在 `ShellPrefs` 和 `get_shell_prefs` 返回中暴露该字段。
   - 新增 `set_overlay_enabled` command，复用现有 `shell.json` 原子保存与备份流程。
   - 添加旧配置缺字段、默认关闭和开关持久化测试。

2. **实现 overlay 窗口管理**
   - 新建 `src-tauri/src/overlay.rs`，集中维护 label、窗口尺寸、创建/显示/隐藏和默认位置计算。
   - 用 `WebviewWindowBuilder` 配置无边框、置顶、跳过任务栏、不可缩放、创建时不聚焦、初始隐藏。
   - 首次显示时定位主显示器工作区右下角；已有持久位置时恢复并校验可见性。
   - 设置开关 command 成功保存后同步显示/隐藏窗口；失败时不改变已持久化状态。
   - overlay 的关闭事件只隐藏，不退出应用；真正退出仍由现有托盘路径负责。

3. **接入窗口位置持久化**
   - 添加并注册官方 window-state 插件及必要 capability。
   - 只保存/恢复 overlay 的位置（尺寸固定，不保存最大化/全屏状态）。
   - 拖动结束或窗口移动事件做节流保存；应用真正退出前确保最终保存。
   - 验证主窗口尺寸、位置和关闭到托盘行为无变化。

4. **新增悬浮条前端入口与 UI**
   - 新建 `OverlayApp.vue`（或等价页面组件），在 `main.ts` 根据窗口 label/hash 选择挂载 overlay 或主应用。
   - 设计紧凑单行状态条：状态点、完整上游模型；次要信息通过第二行或悬停提示展示分组、供应商、最近成功时间。
   - 保持固定响应式尺寸，超长模型名截断但可通过 title/tooltip 查看完整值。
   - 添加拖拽区域；点击打开主窗口的命令使用清晰图标并提供 tooltip。
   - 不加入说明性教程文字，不使用卡片嵌套或装饰性渐变。

5. **实现状态刷新**
   - 复用 `proxy_status` 和 `get_last_success_request`，每 2~3 秒轮询一次；窗口隐藏/卸载时清理定时器。
   - 派生运行且有模型、运行但无模型、停止、异常四类显示状态。
   - 请求失败时保留上次有效数据并显示非阻断错误状态，避免闪烁。
   - 确保故障转移后显示日志中的最终成功上游模型。

6. **设置页接入**
   - `ShellPrefs` TS 类型增加 `overlay_enabled`。
   - 新增 `setOverlayEnabled` API。
   - 设置页增加 `HCheckbox`，沿用现有偏好加载、乐观更新失败回滚、消息提示模式。
   - 开启后立即显示，关闭后立即隐藏，重启后按持久化开关恢复。

7. **托盘与生命周期回归**
   - 验证 main 关闭仍仅隐藏，overlay 不影响单实例唤起。
   - 验证托盘退出会停止代理并关闭两个窗口。
   - 不改请求转发、故障转移和日志写入。

## 自动化验证


重点测试：

- `ShellConfig` 从旧 JSON 加载时 `overlay_enabled == false`。
- 开关保存后重载保持一致。
- overlay 展示状态映射可提取为纯函数时，覆盖有模型、无模型、停止、异常。
- 现有请求日志和最近成功请求测试保持通过。

## Windows 11 手工验收

- [ ] 初次启动悬浮条默认不显示。
- [ ] 设置页开启后立即在主显示器任务栏上方显示，不产生任务栏按钮。
- [ ] 悬浮条出现时不打断记事本/终端等应用的持续输入。
- [ ] 成功请求后 3 秒内更新为最终上游模型；故障转移后显示最终成功候选项。
- [ ] 无成功历史、代理停止、代理异常状态可区分。
- [ ] 可拖动悬浮条，应用重启后恢复位置且仍在可见工作区。
- [ ] 超长模型名不溢出、不遮挡按钮，完整值可查看。
- [ ] 关闭主窗口后悬浮条和代理继续运行；托盘退出后全部关闭并释放端口。
- [ ] 100%、125%、150% DPI 下不被任务栏或屏幕边缘裁切。

## 回滚点

- 若 window-state 插件无法隔离主窗口，回滚插件全局自动恢复，保留只对 overlay 的手动位置持久化。
- 若 Windows 下透明窗口存在渲染或阴影异常，回滚为不透明紧凑状态条，不影响功能验收。
- 若“点击打开主窗口”无法同时满足可点击和绝不抢焦点，MVP 保留拖动与纯展示，移除点击打开命令；主窗口仍从托盘打开。
