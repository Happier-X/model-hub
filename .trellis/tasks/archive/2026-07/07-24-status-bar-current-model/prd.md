# 系统状态栏显示当前模型

## 目标

让 Windows 用户无需打开 Model Hub 主窗口，也能持续、直观地看到当前或最近实际使用的上游模型，减少频繁点击托盘图标确认模型的操作。

## 背景与已确认事实

- 当前应用是 Tauri 2 桌面应用，Windows 为 MVP 验收平台。
- 应用已有系统托盘；关闭主窗口后进程和内嵌代理继续运行。
- 现有托盘只显示固定图标，完整提示仅在悬停时可见，无法满足“直接看到模型”的目标。
- Windows 11 不允许普通桌面应用在任务栏或托盘图标旁永久嵌入任意文本。
- 当前请求日志已保存分组、最终供应商、最终上游模型和时间；后端已有“最近成功请求”查询。
- 当前系统没有独立的实时“正在请求模型”状态；请求完成后只有“最近成功使用的模型”具有稳定语义。

## 需求

- 使用类似桌面悬浮歌词的常驻悬浮状态条，显示在 Windows 任务栏上方，不嵌入原生任务栏。
- 悬浮状态条无边框、置顶且不抢占输入焦点；用户无需显示 Model Hub 主窗口即可看到模型状态。
- 悬浮条默认关闭，由设置页开关控制启用；开关状态持久化。
- 悬浮条可用鼠标拖动改变位置并记住位置；MVP 默认贴主显示器右下角、任务栏上方。
- MVP 仅在主显示器显示悬浮条，不做跨屏跟随，不做全屏自动隐藏检测。
- MVP 只展示最近成功模型，不展示请求进行中的瞬时模型。
- 请求完成后展示最终成功的上游模型，而不是分组配置中的首选模型。
- 发生故障转移时，展示最终实际成功的供应商和上游模型。
- 没有成功请求时展示明确的空状态，不得伪装成某个配置模型。
- 代理停止或异常时，展示状态应与可用模型信息有明显区别。
- 完整信息至少包含：上游模型；辅助信息应包含分组、供应商和最近成功时间。
- 该能力不能要求主窗口常驻任务栏。
- Windows MVP 不依赖已被 Windows 11 弱化或废弃的 DeskBand 等任务栏扩展机制。

## 验收标准

- [ ] 主窗口隐藏后，用户无需点击托盘菜单即可辨认当前展示的模型状态。
- [ ] 一次成功请求结束后，展示在可感知时间内更新为最终实际使用的上游模型。
- [ ] 故障转移成功后，展示的是最终成功候选项，不是失败候选项。
- [ ] 无历史成功请求时显示“暂无模型”或等价空状态。
- [ ] 代理停止和代理异常均有可辨认状态，不会被误认为仍在使用模型。
- [ ] 用户可以查看分组、供应商、完整上游模型和最近成功时间。
- [ ] 隐藏主窗口后代理继续运行，退出行为与现有托盘生命周期保持一致。
- [ ] 悬浮条默认关闭；在设置页开启后显示，关闭后隐藏；开关状态重启后保持。
- [ ] 悬浮条可拖动，重启应用后仍在上次位置。
- [ ] 悬浮条不抢占焦点：显示时正常输入其它应用不受影响。
- [ ] Windows 11 下完成手工验收，文本不被任务栏或屏幕边缘裁切。

## 范围外

- 将自定义文本真正嵌入 Windows 11 原生任务栏。
- macOS 菜单栏和 Linux 面板适配。
- 修改请求路由、模型选择或故障转移策略。
- 展示 API Key、请求内容或其他敏感信息。

## 已决策

- MVP 主展示形态为任务栏上方的常驻悬浮状态条，交互参考桌面悬浮歌词。
- MVP 只展示最近成功模型，不新增请求进行中的瞬时模型状态。
- 悬浮条默认关闭，由设置页开关控制并持久化。
- 悬浮条默认贴主显示器右下角、任务栏上方，可拖动并记住位置。
- MVP 仅在主显示器显示，不做跨屏跟随，不做全屏自动隐藏检测。
