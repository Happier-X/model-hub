# 代理端口占用时自动选择可用端口

## Goal

配置端口被占用时，自动向后寻找可用端口并启动，把实际端口写入 `shell.json`，更新概览 Base URL；不强杀占用进程。

## Requirements

- R1：`proxy_start` / 应用自动启动时，从首选端口起最多尝试 N 个端口（含首选）。
- R2：绑定成功后若端口与配置不同，持久化新端口到 `shell.json`，并在状态中给出中文说明。
- R3：不自动结束占用端口的其他进程。
- R4：范围内全部失败时返回清晰错误（含起止端口）。
- R5：单测覆盖「首选被占 → 选下一可用」。

## Acceptance Criteria

- [x] 首选端口占用时能自动启动在后续端口
- [x] 新端口写入 shell 配置
- [x] 概览展示新 Base URL / 说明
- [x] 不杀占用进程
- [x] cargo test / typecheck 相关通过
