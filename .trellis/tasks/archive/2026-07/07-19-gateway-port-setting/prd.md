# 增加网关端口设置

## Goal

修复“端口被占用时提示用户去设置更换端口，但设置页没有端口入口”的产品缺陷，让用户能够在应用内配置本地网关监听端口并恢复启动。

## Background

- Rust 当前将默认端口固定为 `8080`：`src-tauri/src/gateway/config.rs::DEFAULT_PORT`。
- `GatewayRuntime` 在初始化时持有 host/port，并据此生成侧车配置、环境变量、健康检查地址和前端 `base_url`。
- 设置 UI 集中在 `src/App.tsx`，现有功能只有启停、鉴权、路径等，没有端口输入。
- 端口占用错误目前声称“请在设置中更换端口”，与实际界面不一致。
- 默认绑定必须保持 `127.0.0.1`；不得自动结束占用端口的进程。

## Requirements

### R1. 设置入口

- 设置页新增“网关监听端口”输入项，默认显示当前端口。
- 允许范围 `1–65535`，拒绝空值、小数、非数字和越界值，并展示中文可行动错误。
- 端口占用错误后用户能够进入设置修改并重新启动。

### R2. Rust 配置契约

- 新增 Tauri 命令读取/更新网关端口，或使用一个更新命令返回最新 `GatewayStatus`。
- 端口更新同步影响：侧车配置、`OCTOPUS_SERVER_PORT`、健康检查、状态栏和前端 `base_url`。
- 网关运行期间不得静默切换端口；推荐要求先停止再修改，避免状态与实际进程不一致。
- 默认仍为 `127.0.0.1:8080`。

### R3. 持久化

- 用户修改后的端口应跨应用重启保留，存储于应用配置目录中的壳配置文件；不得依赖浏览器 localStorage 作为唯一真源。
- 配置缺失时使用 `8080`；配置损坏时提供可读错误或安全回退并可恢复。

### R4. 安全与兼容

- 不自动杀占用端口的进程，尤其不得按进程名结束 octopus。
- 不更改网关管理 JWT / 客户端 `sk-octopus-...` 鉴权规则。
- 已有数据目录和内嵌侧车部署逻辑保持兼容。

## Acceptance Criteria

- [x] AC1：设置页可查看、校验并保存端口 `1–65535`。
- [x] AC2：修改端口后启动网关，状态、base URL、侧车配置和健康检查均使用新端口（自动化覆盖 18080 传播链路）。
- [x] AC3：修改值跨应用重启保留；首次运行仍默认 `8080`。
- [x] AC4：运行中修改被明确阻止并提示先停止；不结束任何非托管进程。
- [x] AC5：端口占用错误与设置入口一致；前后端测试、lint/build/check 通过。

## Out of Scope

- 自动扫描并选择空闲端口。
- 配置监听 host 或允许公网绑定。
- 自动结束占用端口的程序。
- 本次立即发布新的 GitHub Release/tag。
