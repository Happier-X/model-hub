# 日志保留最近1000条及最近七天

## 目标

请求日志只保留最近七天内的最新 1000 条，控制本地数据库体积并保持日志查询可用。

## 需求

- 日志保留时间窗口为最近 7 天。
- 日志保留数量上限为最新 1000 条，按日志自增 id 倒序确定最新记录。
- 时间和数量条件同时生效：超过七天或超出最新 1000 条的记录都应被删除。
- 启动、写入日志、打开日志列表时的自动清理继续采用 best-effort，不得阻断请求主流程。
- 日志页展示实际的七天和 1000 条保留策略，手动清理操作使用相同策略。
- 保留现有清空日志、分页、筛选和统计行为。

## 验收标准

- [x] 默认保留天数为 7 天，默认最多保留 1000 条。
- [x] 自动清理会删除七天以前的记录。
- [x] 自动清理会在时间窗口内仅保留最新 1000 条。
- [x] 只满足其中一个条件的记录也会被删除，清理结果和剩余数量准确。
- [x] Rust 单元测试覆盖时间清理、数量清理以及组合清理。
- [x] 前端日志页不再显示旧的 30 天策略。
- [x] `cargo fmt --check`、`cargo test` 和 `cargo check` 通过。
