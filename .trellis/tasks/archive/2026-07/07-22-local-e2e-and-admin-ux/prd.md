# 本机端到端验收与管理端体验补齐

## Goal

在现有 Vue3 管理端 + Tauri 内嵌代理已具备核心能力的前提下，补齐本机验收路径上的**明显管理端体验缺口**，并沉淀一份可重复的本机验收清单，降低「配好了但不好用/看不懂状态」的成本。

## 已确认事实（仓库审计）

已有页面：概览（启停/端口/curl）、供应商、分组（队列+自动故障转移）、API Key（一次明文）、日志（含 failover 字段）、健康接口 `list_health`（含 `consecutive_failures`）。

### 体验缺口（代码证据）

| 缺口 | 证据 | 影响 |
|------|------|------|
| 健康状态弱展示 | Providers/Groups 仅文字「健康/警告/熔断/半开」，**未展示** `consecutive_failures` | 难判断是否接近熔断 |
| 无自动刷新 | 各页 `onMounted(refresh)`，健康/状态需手动刷新 | 联调故障转移时状态滞后 |
| 概览引导偏薄 | 有 curl 示例，但缺「推荐配置顺序」清单 | 新用户不知先建 Key/供应商/分组 |
| 日志状态可读性 | 状态码纯数字，无成功/失败视觉区分 | 扫日志慢 |
| 端到端清单缺失 | 无仓库内一页式本机验收步骤 | 回归靠记忆 |

### 后端/代理

故障转移、静默超时日志、缺列迁移等近期已修；本任务**默认不改**代理协议，除非验收清单强制要求的纯展示字段。

## Requirements（MVP 建议）

- R1：供应商列表健康列展示 **状态徽章 + 连续失败次数**（数据来自 `list_health`）。
- R2：分组队列条目同样展示供应商健康徽章与连续失败次数。
- R3：概览页增加简短 **本机接入步骤**（启动代理 → 建供应商 → 建分组队列 → 建 Key → curl/客户端），并保留 Base URL 复制与示例 curl。
- R4：日志页状态码做基础样式区分（2xx / 4xx / 5xx / 其他）。
- R5：在任务产物或 `docs/` 增加 **本机验收清单**（可勾选步骤，含故障转移与 Key 鉴权）。
- R6（可选增强）：概览或供应商页提供「刷新健康」按钮；**不做**复杂轮询除非验收明确需要。

## Acceptance Criteria

- [x] 供应商页可见健康状态与连续失败次数。
- [x] 分组页队列条目可见对应供应商健康信息。
- [x] 概览页有清晰接入步骤与可复制 Base URL / curl。
- [x] 日志页状态码有基础视觉区分。
- [x] 存在可执行的本机验收清单文档。
- [x] `pnpm typecheck`、`pnpm build`、`pnpm lint` 通过（或项目既有 lint 基线不恶化）。

## Out of Scope

- 重做视觉设计系统 / 暗色主题。
- 高级熔断参数配置 UI。
- 自动探测上游 models 列表。
- 修改故障转移/熔断后端默认参数。

## 已确认决策

1. 健康数据：**仅进页刷新 + 手动刷新按钮**，不做定时轮询。
2. 验收清单：写入 **`docs/local-acceptance.md`**。
