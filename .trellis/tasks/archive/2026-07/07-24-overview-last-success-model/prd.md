# 概览展示最近成功模型

## Goal

在概览页展示**全局最近一次成功请求**实际用到的模型信息，帮助用户一眼看到「当前在用」的上游（日志态，而非配置队列首选）。

## Requirements

- R1：概览页新增一块展示「最近成功请求」：至少包含 **分组名**、**供应商名**、**上游模型**、**请求时间**。
- R2：「成功」定义与现有今日统计一致：`status_code` 在 200–299 且 `error` 为空（或空串）。
- R3：粒度固定为 **全局一条**（全库日志中满足成功条件、`time` 最大的一条）；不按分组列表展开。
- R4：无成功日志时展示明确空态（例如「暂无成功请求」），不报错、不阻塞其它概览区块。
- R5：进入概览与刷新统计时同步刷新该信息；失败时展示可读错误，不影响代理启停等其它操作。
- R6：后端通过 Tauri command 提供专用查询（或与现有 stats 同级的明确接口），避免前端拉整页日志再筛。

## Out of Scope

- 不展示实时在途（in-flight）请求。
- 不按分组各展示一条。
- 不改故障转移策略、不写新业务日志字段。
- 不自动轮询刷新（除非现有概览已有轮询；本任务不新增定时器）。

## Acceptance Criteria

- [ ] 存在至少一条成功日志时，概览显示其分组 / 供应商 / 上游模型 / 时间。
- [ ] 多条成功日志时，显示 `time` 最新的一条。
- [ ] 仅有失败日志或无日志时，显示空态文案。
- [ ] 「成功」判定与 `RequestStats.success` 一致（2xx 且无 error）。
- [ ] 刷新统计或重新进入概览后数据会更新。
- [ ] 相关单元/集成测试覆盖查询边界；前端类型与 UI 通过 typecheck/lint。
