# PRD：网关启动时自动打开与改端口自动重启

## 背景

用户期望：打开应用后网关默认已运行；在设置中修改监听端口并保存后，应自动按新端口重启，无需先手动停止。

当前：
- 启动时已有 `try_autostart`，但失败时错误可能不显眼。
- `gateway_set_port` 在 running/starting/stopping 时直接拒绝（`PortChangeWhileActive`）。
- 设置页 UI 运行中禁用端口输入。

## 目标

1. **打开应用默认启动网关**（保持/强化现有自动启动；无开关）。
2. **保存端口后自动 stop → 写配置 → start**；成功则状态为 running + 新端口。

## 范围

### 必须

- 壳：`gateway_set_port` 在网关活跃时先 `stop`，再改端口、持久化 `shell.json`，再 `start`
- 前端：允许运行中改端口；文案改为「保存后自动重启」
- 文档/设置说明同步
- 单测覆盖：活跃态改端口不再返回 `PortChangeWhileActive`（或由更高层保证）

### 不做

- 不做「启动时自动启动」开关
- 不自动结束非本应用托管的占用端口进程
- 本任务可不单独发版（用户可再要求）

## 验收标准

- [x] 冷启动应用后，正常情况下网关进入 running（或错误可在设置/状态条看到）
- [x] 运行中保存新端口后，无需手动停止；状态变为新端口 running
- [x] UI 不再要求「请先停止网关再改端口」
- [x] cargo test / pnpm lint 通过
