# 修复应用退出后端口仍占用

## Goal

应用**真正退出**后本机代理端口必须释放；关窗仍隐藏到托盘且代理继续运行。消除 `stop` 超时后任务仍占端口，并引入**轻量单实例**，避免多开互相占端口。用户能分清「隐藏」与「退出」。

## Decisions

| 项 | 结论 |
|----|------|
| 关窗 | 隐藏托盘，代理继续（方案 A） |
| 单实例 | **做**（方案 A）：第二开聚焦已有窗口/托盘，不启第二套代理 |
| 外部进程 | 不 kill 无关占用者 |

## Requirements

- R1：真正退出（托盘「退出」、`RunEvent::Exit`）可靠停止代理并释放监听端口。
- R2：`stop()` graceful 等待有限时间；**超时必须 abort** 服务任务，禁止丢弃 JoinHandle 后仍占端口。
- R3：`ProxyHandle` Drop 时 best-effort `stop`。
- R4：托盘 tooltip / 概览等文案：关窗 = 隐藏到托盘、代理仍运行；仅「退出」停止代理。
- R5：不强制 kill 外部无关进程；自动改口可保留，文案需可理解。
- R6：**单实例**：同时仅一个 Model Hub 桌面进程；第二实例通知第一实例显示窗口后退出，且不 `proxy.start`。
- R7：关窗隐藏行为保持不变。

## Out of Scope

- 关窗即退出。
- 主动结束占用端口的第三方进程。
- 改默认 host / 端口扫描上限（除非常量微调）。

## Acceptance Criteria

- [ ] 托盘「退出」后原端口短时间内可重新 bind
- [ ] stop 超时后 abort，无长期占用
- [ ] 第二开不启第二代理，聚焦已有实例
- [ ] 关窗后托盘可恢复窗口，代理仍可运行
- [ ] 文案区分隐藏与退出
- [ ] cargo test / typecheck / lint 通过
