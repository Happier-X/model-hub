# 新建供应商和分组使用对话框

## Goal

将供应商页和分组页当前常驻页面顶部的新建表单改为按需打开的对话框，让列表成为页面主体并减少表单长期占用空间。

## Background

- `src/pages/ProvidersPage.vue` 当前将供应商创建/编辑表单常驻在列表上方。
- `src/pages/GroupsPage.vue` 当前将分组创建/编辑及故障转移队列表单常驻在列表上方。
- 项目当前未引入 Dialog 组件库，也没有通用 Dialog 组件。
- 两个页面的编辑动作目前复用各自的新建表单状态。

## Requirements

- R1：供应商页提供明确的「新建供应商」入口，点击后打开对话框。
- R2：分组页提供明确的「新建分组」入口，点击后打开对话框。
- R3：供应商和分组的编辑操作复用各自的新建对话框，以稳定 id 区分新建/编辑模式，不保留页面内嵌编辑表单。
- R4：对话框关闭时不得误提交；再次新建应获得干净的默认表单。
- R5：保存失败时保留对话框及用户输入，便于修正后重试。
- R6：保存成功后关闭对话框并刷新列表。
- R7：分组页「拉取模型」仍仅由用户点击触发，禁止打开对话框时自动访问上游。
- R8：对话框应具备基础可访问性：语义化标题、遮罩、关闭按钮、Escape 关闭及合理的键盘焦点行为。
- R9：点击遮罩、关闭按钮、取消或按 Escape 时直接关闭并重置表单，不增加未保存修改二次确认。
- R10：供应商对话框使用普通宽度；分组对话框使用接近视口宽度的大尺寸，最大高度约为视口的 90%，超出内容在对话框内部滚动。

## Acceptance Criteria

- [ ] 供应商页初始只展示列表和「新建供应商」按钮，不常驻显示创建表单。
- [ ] 点击「新建供应商」打开供应商表单对话框，可成功创建并自动关闭；点击「编辑」在同一对话框加载并更新原供应商。
- [ ] 分组页初始只展示列表和「新建分组」按钮，不常驻显示创建表单。
- [ ] 点击「新建分组」打开完整分组表单对话框，可成功创建并自动关闭；点击「编辑」在同一对话框加载并更新原分组。
- [ ] 点击遮罩、关闭按钮、取消或按 Escape 可直接关闭；重新新建时表单恢复默认值。
- [ ] 保存失败时对话框不关闭且输入不丢失。
- [ ] 供应商对话框宽度适合常规表单；分组对话框在桌面端接近视口宽度、内容过高时仅在对话框内部滚动。
- [ ] 打开分组对话框不会自动拉取供应商模型。
- [ ] `pnpm test:unit`、`pnpm typecheck`、`pnpm lint` 通过。

## Out of Scope

- 修改后端供应商或分组 CRUD 接口。
- 引入完整 UI 组件库。
- 修改删除确认交互。
