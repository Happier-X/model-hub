# 首页功能拆分到设置

## Goal

新增独立的「设置」页面，将首页中的低频配置型功能迁移过去，使首页聚焦代理运行状态、请求概况和日常操作。

## Confirmed Facts

- 当前前端路由包含首页、供应商、分组、日志，尚无设置页。
- 当前首页包含：
  - 今日请求统计与最近成功请求
  - 本地代理状态、Base URL、监听地址、数据目录
  - 端口修改、代理启动/停止、状态刷新
  - 应用更新检查、下载安装、进入首页时自动检查更新偏好
  - 本机接入步骤和 curl 调用示例
- 代理配置和应用偏好均通过 `src/api/tauri.ts` 的现有 invoke 接口读写，不需要新增前端直连请求。

## Scope

### 首页保留

- 今日请求统计与最近成功请求
- 代理运行状态、Base URL、监听地址
- 复制 Base URL、启动、停止、刷新等日常操作
- 本机接入步骤和 curl 调用示例（属于使用指引，不是设置）

### 设置页迁入

- 代理端口修改
- 数据目录展示
- 应用更新检查、下载安装与当前版本展示
- 自动检查更新偏好

## Requirements

- 新增侧边栏「设置」入口和 `/settings` 路由。
- 新增 `SettingsPage.vue`，承载最终确认的配置型功能。
- 首页删除已迁移功能及其不再需要的状态、方法和 import。
- 功能迁移后行为保持不变，包括错误提示、加载态、更新资源释放和安装重启流程。
- 自动检查更新改为「应用启动时自动检查一次」，设置项文案更新为「应用启动时自动检查更新（仍需确认后才安装）」；启动检查逻辑从首页生命周期提升到应用壳层或专用组合式函数，避免路由切换重复触发。若发现新版本，在应用壳顶部显示可关闭的轻量提示，并提供「前往设置」入口；启动检查只探测并释放更新资源，下载和安装仍在设置页手动触发。
- 同步更新相关 README、docs 和 `.trellis/spec/` 信息架构及页面职责。

## Acceptance Criteria

- [ ] 侧边栏可进入设置页，页面标题显示「设置」，刷新 `/settings` 可正常加载。
- [ ] 首页只保留最终确认的状态、日常操作、统计和使用指引。
- [ ] 设置页可读取并保存代理端口，保存后的行为与当前一致。
- [ ] 设置页可检查更新、确认下载安装、取消待处理更新，并配置自动检查偏好。
- [ ] 开启自动检查偏好后，应用启动时自动检查一次更新，路由切换不重复触发；发现新版本时显示可关闭提示并可跳转设置页。
- [ ] 页面拆分后无重复状态源或重复实现同一业务流程。
- [ ] `npm run typecheck`、`npm run lint` 和相关测试通过。
- [ ] 文档与 Trellis spec 中页面职责和导航信息同步更新。

## Resolved Decisions

- 迁移范围采用方案 A：端口、数据目录、更新面板、自动检查偏好迁入设置；统计、代理状态与启停、接入指引留首页。
- 自动检查更新时机：应用启动时自动检查一次。
- 启动检查发现新版本时，应用壳顶部显示可关闭的轻量提示，点击「前往设置」进入设置页；启动检查不持有跨页面 `Update` 资源，不执行下载或安装。

## Out of Scope

- 修改后端配置结构或数据库。
- 新增供应商、分组、日志相关设置。
- 新增独立帮助/文档页面，除非后续明确扩大范围。
