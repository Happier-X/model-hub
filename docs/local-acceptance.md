# 本机验收清单

面向本机桌面端（Vue3 管理台 + Tauri 内嵌代理）的可勾选回归步骤。  
自动化门禁仍见 [mvp-acceptance.md](./mvp-acceptance.md)；本页侧重**手工联调路径**与故障转移/健康可读性。

## 前置

- [ ] 已安装依赖：`pnpm install`
- [ ] 前端门禁通过：`pnpm lint`、`pnpm typecheck`、`pnpm build`
- [ ] 可选：`cd src-tauri && cargo test && cargo check`
- [ ] 以开发或打包方式启动应用（例如 `pnpm tauri dev`）

## 0. 首页与代理生命周期

- [ ] 打开 **首页**，可见状态、Base URL、监听地址
- [ ] 页面有「本机接入步骤」说明（启动代理 → 供应商 → 分组 → curl）
- [ ] 可复制 Base URL；示例 curl 中 `model` 为分组名，**无**客户端 Key 要求
- [ ] 首页启停正常；`running` / `idle` / `error` 状态可读
- [ ] 打开 **设置** 页，可见当前监听、数据目录；修改端口并保存后行为与首页启停一致
- [ ] 设置页可检查更新、配置「应用启动时自动检查更新」
- [ ] 代理未运行时状态明确，最后错误（如有）可行动
- [ ] 导航中**无**「API Key / API 密钥」入口

## 1. 供应商

- [ ] 新建至少两个供应商（建议：一个故意错误的 Base URL/Key，一个可用 OpenAI 兼容上游）
- [ ] 列表展示名称、Base URL、启用状态
- [ ] **健康列**显示状态徽章（健康 / 警告 / 熔断 / 半开）与 **连续失败次数**
- [ ] 点击「刷新健康」后健康数据更新（无定时轮询，进入页面也会刷新）
- [ ] 编辑、启停、删除可用

## 2. 分组（队列 + 故障转移）

- [ ] 新建分组，分组名即客户端 `model`
- [ ] 队列至少两条：第一条指向失败上游，第二条指向可用上游；开启自动故障转移
- [ ] 列表中队列条目展示供应商名、上游模型、**健康徽章 + 连续失败次数**
- [ ] 「刷新健康」可用；上移/下移后顺序保持

## 3. 本机访问（无客户端鉴权）

- [ ] 无 `Authorization` 可调用：
  ```bash
  curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:<端口>/v1/models
  # 期望 200（已有分组时）
  ```
- [ ] 带任意 Bearer **不**返回 401：
  ```bash
  curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:<端口>/v1/models \
    -H "Authorization: Bearer arbitrary-value"
  # 期望 200
  ```

## 4. Models 与 Chat

- [ ] `GET /v1/models` 返回的 `id` 为分组名（非上游原始模型名混入）
- [ ] 非流式 Chat（`model` = 分组名）成功返回：
  ```bash
  curl http://127.0.0.1:<端口>/v1/chat/completions \
    -H "Content-Type: application/json" \
    -d '{"model":"<分组名>","messages":[{"role":"user","content":"hi"}]}'
  ```
- [ ] 可选：流式 `stream: true` 能正常输出；静默超时等失败在日志中为**单条**结论（无「先 200 再 504」双结论）

## 5. 故障转移与熔断健康

- [ ] 开启自动故障转移：首供应商失败、次供应商成功 → 请求最终成功
- [ ] **日志**可见 `failover_from` / `failover_to` / `failover_reason`
- [ ] 关闭自动故障转移后，首供应商失败时**不会**尝试第二条
- [ ] 同一供应商连续失败达熔断阈值后，供应商/分组页健康为 **熔断**，连续失败次数 ≥ 阈值
- [ ] 等待恢复窗口后可为 **半开**；探测成功后回到健康、失败次数清零（以实际熔断实现为准）

## 6. 日志页可读性

- [ ] 可见：时间、分组、供应商、上游模型、状态码、耗时、错误、故障转移字段
- [ ] 状态码有基础视觉区分：**2xx 绿 / 4xx 琥珀 / 5xx 红 / 其他灰**
- [ ] 日志不含完整上游 Key 或 messages 正文
- [ ] 清空日志可用

## 7. 安全与边界速查

- [ ] 默认监听 `127.0.0.1`
- [ ] 上游 Key 不在日志/列表完整泄露（供应商表单内编辑除外）

## 推荐最小联调路径（约 10 分钟）

1. 首页启动代理，复制 Base URL  
2. 建两个供应商 + 一个双队列分组（自动故障转移开）  
3. 无 Key 调用 `models` 与 Chat 成功；带任意 Bearer 也不 401  
4. Chat 走故障转移成功，日志核对转移字段  
5. 供应商/分组页核对健康徽章与连续失败次数，手动刷新健康  

完成上述勾选即可视为本机端到端体验验收通过。
